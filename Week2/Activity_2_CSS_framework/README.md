# Activity 2: CSS Framework Styling

## Objective
Learn how to use CSS frameworks to quickly style web pages by applying a framework of your choice to the provided `index.html` file.

## Instructions

1. **Choose a CSS Framework** from the list below
2. **Research** the framework's documentation to understand how to use it
3. **Link** the framework to your `index.html` file (usually via CDN link in the `<head>` section)
4. **Apply** framework classes to the HTML elements to style the page
5. **Experiment** with different components and utilities provided by the framework

## Popular CSS Frameworks

Choose one of the following CSS frameworks to complete this activity:

- **Bootstrap** - The most popular CSS framework for building responsive, mobile-first websites
  - URL: [https://getbootstrap.com/](https://getbootstrap.com/)
  - Documentation: [https://getbootstrap.com/docs/](https://getbootstrap.com/docs/)

- **Tailwind CSS** - A utility-first CSS framework for rapidly building custom designs
  - URL: [https://tailwindcss.com/](https://tailwindcss.com/)
  - Documentation: [https://tailwindcss.com/docs](https://tailwindcss.com/docs)
  - CDN: [https://tailwindcss.com/docs/installation/play-cdn](https://tailwindcss.com/docs/installation/play-cdn)

- **Bulma** - A modern CSS framework based on Flexbox
  - URL: [https://bulma.io/](https://bulma.io/)
  - Documentation: [https://bulma.io/documentation/](https://bulma.io/documentation/)

- **Foundation** - A responsive front-end framework by ZURB
  - URL: [https://get.foundation/](https://get.foundation/)
  - Documentation: [https://get.foundation/sites/docs/](https://get.foundation/sites/docs/)

- **Materialize** - A modern responsive front-end framework based on Material Design
  - URL: [https://materializecss.com/](https://materializecss.com/)
  - Documentation: [https://materializecss.com/getting-started.html](https://materializecss.com/getting-started.html)

- **Semantic UI** - User interface framework designed around natural language
  - URL: [https://semantic-ui.com/](https://semantic-ui.com/)
  - Documentation: [https://semantic-ui.com/introduction/getting-started.html](https://semantic-ui.com/introduction/getting-started.html)

- **Pure CSS** - A set of small, responsive CSS modules by Yahoo
  - URL: [https://purecss.io/](https://purecss.io/)
  - Documentation: [https://purecss.io/start/](https://purecss.io/start/)

- **UIKit** - A lightweight and modular front-end framework
  - URL: [https://getuikit.com/](https://getuikit.com/)
  - Documentation: [https://getuikit.com/docs/introduction](https://getuikit.com/docs/introduction)

## Tips

- Start by adding the framework's CSS (and JavaScript if needed) via CDN links in the `<head>` section
- Review the framework's documentation for component examples (buttons, navigation, forms, cards, etc.)
- Apply the appropriate classes to the HTML elements in `index.html`
- Focus on making the page responsive and visually appealing
- Experiment with the framework's grid system to create layouts
- Don't be afraid to combine multiple components from the framework

## Deliverables

- A styled `index.html` file using your chosen CSS framework
- The page should be responsive and visually improved compared to the unstyled version
