# Activity 1 — Semantic Fix‑Up Sprint (15–20 min)

PROG 2700 Client-Side Programming

## Objectives

- Identify and correct non‑semantic markup
- Use proper form labels and structure

## Materials

- `starter.html` (intentionally flawed)
- Browser + DevTools

## Setup

1. Open `starter.html` in your browser.
2. Open DevTools → **Elements** tab.

## Guided Steps

1. **Scan (2 min):** List non‑semantic tags/usages you notice.
2. **Replace:** Convert generic `<div>` blocks into semantic sections: `header`, `main`, `footer`, `nav`, `section`, `article`, and `form`.
3. **Labels:** Ensure each input has an associated `<label for="...">` whose value matches the input `id`.
4. **Validate:** Run the page through the W3C validator (https://validator.w3.org/) and note at least **one error** to fix.

## Teacher Prompts (ask live)

- Why does semantic structure help accessibility?
- How does `<label>` improve focus/UX?

## Wrap‑Up

- Share one change that improved readability.
- Commit your changes to your repo (or save your changes locally or on OneDrive).
